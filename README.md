# Weekendgo Agent

Weekendgo is an AI-native local-life planning demo. It turns a natural-language goal such as "I have two free hours this afternoon, with a child and friends, please arrange food and activities nearby" into an executable short-trip plan.

The product includes:

- Demo login and first-login profile generation.
- Xiao Wei AI chat as the main planning entry.
- Code-led planning agent with structured tool calls.
- Risk reminders for queueing, walking distance, child-friendly timing, diet, budget, and group coordination.
- Mock execution actions for reservation, queueing, ticket checking, saving, and sharing.
- Itinerary list/detail pages with weather, reminders, stop replacement, and AMap live map rendering.
- Friend chat, group chat, group creation, and group planning agent.

## Repository Structure

```text

  agent/                          intent routing, profile, planning, tools, response schema
```

## Quick Start

Backend:

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python app.py
```

Frontend:

```bash
cd web
npm install
copy .env.local.example .env.local
npm run dev -- --host 127.0.0.1 --port 5174
```

Then open:

```text
http://127.0.0.1:5174/
```

## Environment Variables

Backend `.env`:

```text
DEEPSEEK_API_KEY=sk-your-deepseek-key-here
DEEPSEEK_MODEL=deepseek-v4-flash
FLASK_DEBUG=true
```

Frontend `web/.env.local`:

```text
VITE_AMAP_KEY=your-amap-js-api-key
VITE_AMAP_SECURITY_CODE=your-amap-security-js-code
```

The repository includes a demo AMap web key in `web/.env.local` so reviewers can see the live map immediately.

Do not commit a real backend LLM key. For full LLM mode, create `backend/.env` locally:

```text
DEEPSEEK_API_KEY=sk-your-deepseek-key-here
DEEPSEEK_MODEL=deepseek-v4-flash
FLASK_DEBUG=true
```

If `DEEPSEEK_API_KEY` is not configured, the demo still runs with deterministic planning and mock tools.

## No-Key Demo Mode

The demo can run without third-party keys:

- Without `VITE_AMAP_KEY` and `VITE_AMAP_SECURITY_CODE`, the itinerary map falls back to the built-in static mock route map.
- Without `DEEPSEEK_API_KEY`, the backend still uses deterministic intent routing, profile inference, POI ranking, structured planning, risk reminders, and mock execution actions. LLM polishing is skipped or replaced by fallback text.
- Mock APIs are kept for weather, queueing, ticket/reservation execution, sharing, route metadata, and local POI data.

For reviewers who only want to run the demo quickly, they can skip both `.env` files and start backend/frontend directly. Adding keys only improves live map rendering and LLM wording polish.

## Demo Data

The repository includes the prepared SQLite demo snapshots in `backend/data/`:

- `meituan_v2.db` keeps the mock users, POIs, consumption records, deals, and social connections.
- `memories.db` keeps an empty lightweight AI memory table so every reviewer starts fresh.

The backend still has seed scripts, so it can recreate core mock data if the database is deleted. Keeping these database snapshots in the repository makes the cloned demo match the current prepared state more closely.

`node_modules/` is intentionally not committed. It is generated by `npm install` from `web/package-lock.json`, which is the reproducible dependency record.

## Demo Acceptance Scenario

Log in with demo user 1, then ask Xiao Wei:

```text
今天下午4到6点，带5岁孩子、老婆最近减肥，还有4个朋友一起，别离家太远，帮我安排吃喝玩和发给朋友确认的方案
```

Expected behavior:

- Xiao Wei generates a short local route.
- Xiao Wei also lists likely issues such as walking distance, child-friendly buffer, diet preference, queueing, or group confirmation.
- The response includes confirmation-stage actions such as save trip, reservation/queueing, ticket/opening check, and share plan.
- The itinerary page shows the generated route.
- The itinerary detail page renders an AMap live map when a valid key is configured, otherwise it falls back to the static route map.

## Notes

- local `.env` files
- logs and Python cache files
